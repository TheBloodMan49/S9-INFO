Déceler les secrets du hash

Le Gopher doit briser les empreintes pour retrouver les secrets cachés

L’histoire

Le petit Gopher est maintenant face à des mystérieux hash bcrypt. Le secret est dissimulé dans une wordlist, mais il ne sait pas s’il doit avancer pas à pas ou mobiliser une armée de travailleurs pour accélérer sa quête.

$2a$10$fMseHU97hUrALqerox/xluLxU2L7aI9zY5HW94jmJxnvTdMz.uWGu

$2a$10$dJq4XBcA/CfnKwORyroj7e86RY5UbwkTomWzSo4AHnMTO7kdh7rae

Le défi

Implémente un programme en Go pour casser un mot de passe dont le hash bcrypt est fourni. Compare deux approches :

  • Version séquentielle : tester chaque mot de passe un par un.
  • Version concurrente : utiliser des goroutines et des workers pour paralléliser le cassage.

Trouve le secret caché dans rockyou.txt (séquentiel) et celui présent dans 10k-most-common.txt (parallélisé).

Rappel

Fonction de hashage : transforme une donnée en une empreinte unique appelée hash.

  • Irréversible : impossible de retrouver le mot original.
  • Déterministe : le même mot donne toujours le même hash.
  • Rapide à calculer, mais difficile à inverser.

bcrypt : conçu pour les mots de passe.

  • Lent volontairement pour rendre le bruteforce coûteux.
  • Inclut un salt pour éviter les attaques par tables arc-en-ciel.
  • Couramment utilisé pour sécuriser les mots de passe en base de données.

Objectifs

2-1.go → Version séquentielle :

Lire le hash cible et la wordlist.
  Tester chaque mot séquentiellement.
  Comparer son hash bcrypt avec le hash cible.
  Afficher le mot trouvé et le temps d’exécution.

2-2.go → Version concurrente :

Créer un pool de workers avec des goroutines.
  Distribuer les mots de passe via un channel.
  Synchroniser avec sync.WaitGroup.
  Afficher le mot trouvé et le temps d’exécution.

Bonus :

  • Limiter le nombre de goroutines avec un pool.
  • Ajouter un timer pour mesurer la vitesse entre les 2 versions.
  • Tester l’impact du nombre de workers.
  • Réfléchir aux risques d’un trop grand nombre de goroutines.

Entrées et sorties

  • Entrée : hash cible + fichier wordlist.txt.
  • Sortie : mot de passe trouvé (ou message si absent) + temps total d’exécution.

Exécution

Version séquentielle :

go run 2-1.go '$2a$10$fMseHU97hUrALqerox/xluLxU2L7aI9zY5HW94jmJxnvTdMz.uWGu' rockyou.txt

Version concurrente :

go run 2-2.go '$2a$10$rne/oP9aH7CgPbkhI..ZvOBhzePQkhW4s8gbpaTeeJG9nZnb0EtwG' 10k-most-common.txt

Conseil du Gopher

"Les secrets ne se révèlent qu’à ceux qui savent comparer les empreintes… et orchestrer leurs workers."

Bonne chance, Gopher. Les hashs ne cèdent qu’à ceux qui décryptent avec méthode.
